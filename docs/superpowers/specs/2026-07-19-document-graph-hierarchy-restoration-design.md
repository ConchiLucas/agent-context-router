# 文档层级关系图恢复设计

## 目标

Documents 的 Graph 视图恢复原版“入口 → 分支 → 下一层文档”的连接线层级表达，不再把主文档树展示为纵向 Level 分组列表。同时保留当前同步得到的任意深度、孤立文档和断链信息。

## 主树布局

- 根节点固定使用当前 Project 中 `graph_depth=1`、`doc_type=agent_index` 的 `AGENTS.md`。
- 从根节点开始，按照有效 `DocumentLink` 递归构建树；每条边保留 Markdown 链接 label。
- 每一层从左向右展开，并使用原版连接线、箭头、节点卡片和预览入口。
- 深度不限制为三层；第四层及以后继续使用相同的“连接线 + 子节点列”结构。
- 同一文档被多个父节点引用时只在最短深度首次展开，其他位置显示为引用节点，避免循环或无限重复。
- 图数据来自同步后的 `is_reachable`、`graph_depth` 和 links，不能推测不存在的父子关系。

## 辅助健康区域

- `is_reachable=false` 的文档不混入主树，继续在主树下方显示为 Orphan documents。
- `is_broken=true` 的链接不生成可点击子节点，继续在独立 Broken links 区域显示 source、label 和 target path。
- List 视图继续保留 Reachability、Depth、Broken links 列，不受本次布局恢复影响。

## 组件边界

- `document-health.ts` 增加从文档集合构建安全递归树的纯函数，负责排序、循环保护和重复引用标记。
- `document-graph.tsx` 只负责递归渲染树、连接线、孤立文档和断链。
- CSS 恢复原版节点视觉，并增加可水平滚动的任意深度容器；窄屏下按层级纵向降级，不能截断节点。

## 验证

- 纯函数测试覆盖 1→2→3→4 层、分叉、循环和同一目标被多个父节点引用。
- 前端 test、lint、build 全部通过。
- 浏览器确认入口、第二层、第三层和更深节点通过连接线形成一棵树；孤立文档和断链仍在主树下方展示。

## 不在本次范围

- 不改变后端同步、MCP read 权限或数据库图深度计算。
- 不改成可拖拽、缩放的画布图。
- 不恢复旧版固定三层限制。
